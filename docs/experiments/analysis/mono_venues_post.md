# Игра в одиночку: что дают моноплощадки в синхронах

*Черновик поста по данным rating API + модели ChGK. Синхроны, офлайн-площадки (без «Онлайн»).*

## Что такое моноплощадка

В синхроне одна и та же игра в разных городах — это один турнир в базе, но **разные площадки** (заявки `tournament_synch_requests`). Если на площадке в этот тур **одна команда**, мы считаем это **моноплощадкой** (`teams_played = 1`). Если две и больше — **обычная** (multi).

Это не «другой пак» и не другой город сам по себе: та же «Баку» в одном туре может быть mono для одной команды и multi для десяти других.

## Остаток и фильтр по опыту команды

Смотрим **остаток** относительно модели: `факт − ожидание` (вопросов за тур; ожидание — по θ **до тура**).

У команд с малым числом игр оценка силы состава **шумная** — ожидание и остаток гуляют. Ниже основные цифры для команд с **≥ 50 синхронов** с известной площадкой (~2 000 команд, ~317 000 игр). Для сравнения — сноска «без фильтра».

---

## Сколько лучше играют одни? (команды с ≥ 50 игр)

| | Моноплощадка | Обычная площадка |
|---|:---:|:---:|
| Игр команд | **24 717** (7,8%) | 292 285 |
| **Медиана** (факт − ожидание) | **+0,47** | **+0,41** |
| Медиана взятий | **19** | 18 |
| Медиана ожидания | 18,0 | 17,3 |

**Разница медиан остатка: +0,06 вопроса** — почти ничего.

На mono типичная команда **и так** чуть переигрывает модель (+0,47); на multi у «опытных» команд картина **такая же** (+0,41), а не минус, как в полной выборке (−0,16). То есть эффект «mono лучше зала» на **зрелых** командах — в основном **артефакт смеси** с тысячами малоиграющих, у которых θ ещё не сошёлся.

| Без фильтра (все команды) | mono | multi | lift |
|---------------------------|:----:|:-----:|:----:|
| Медиана остатка | +0,28 | −0,16 | **+0,44** |

---

## Внутри одной команды (и mono, и multi, ≥ 50 игр всего)

У **1 071** команды с ≥ 50 sync-игр есть и mono, и multi. **Личный lift** = медиана(остаток на mono) − медиана(на multi).

| Когорта | Команд | Медиана lift | Доля с lift > 0 |
|---------|:------:|:------------:|:-----------------:|
| ≥ 50 игр всего, ≥ 1 mono и ≥ 1 multi | 1 071 | **−0,06** | 48% |
| те же, **≥ 5 mono и ≥ 5 multi** | 612 | **≈ 0** | ~50% |

**Типичная опытная команда на моноплощадке не играет лучше, чем в общем зале** — если смотреть честно, внутри себя.

Для сравнения, без порога по 50 играм: медиана lift **+0,15** (2 354 команды) — снова завышается за счёт «малосыгранных».

---

## Кто играет только «в одиночку»

Среди команд с ≥ 50 играми по-прежнему есть те, кто почти всегда на mono (мало multi). Они **не попадают** во внутрикомандное сравнение — но и **не тянут** среднее вверх, если их отрезать вместе с остальными малоиграющими.

---

## Выводы для поста

1. **Громкий +0,4 вопроса «mono vs зал»** — в основном про **всю базу** с новичками и редкими выходами; у команд с **≥ 50** sync-игр остаток **+0,06** (популяция) и **≈ 0** (внутри команды).
2. На mono по-прежнему **выше абсолютный счёт** (медиана 19 vs 18), но модель это **в основном уже учитывает** (ожидание 18,0 vs 17,3).
3. Истории про «жуликов» и аномальные lift +10 — по-прежнему про **узкие** срезы (мало multi, шумная θ), а не про закономерность для зрелых команд.
4. **М4А1** и похожие бакинские регуляры — отдельный кейс: много игр в обоих форматах, умеренный личный lift, переигрыш модели **и там, и там**.

---

## Методика

- Площадки: API `api.rating.chgk.info` → `data/venue_overlay.duckdb`.
- Ожидание: модель ChGK, θ до тура (`chgk.duckdb`).
- Синхроны, без площадок «Онлайн».
- Фильтр **≥ 50** sync-игр с площадкой на команду (сумма mono + multi).

Пересчёт:

```bash
python scripts/analyse_venue_effects.py --min-team-games 50
python scripts/analyse_venue_mono_contrast.py --min-team-games 50 --min-mono 1 --min-multi 1
python scripts/analyse_venue_suspects.py --min-team-games 50
```
